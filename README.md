# saas-crm-user-access-review-control-mapping lab
## 1. Project Overview
This project simulates a GRC control-mapping exercise following a failed audit of user access controls for a SaaS CRM platform.

## 2. Business Scenario
A UK organisation failed an internal audit because it could not evidence that CRM access was approved, reviewed, removed or monitored effectively.

## 3. System in Scope
- SaaS CRM
- Microsoft Entra ID
- HR system
- ServiceNow / Jira Service Management
- CRM audit logs
- Supplier support access

## 4. Control Objectives
- Map risks to NIST SP 800-53 controls
- Identify control owners
- Define evidence requirements
- Assess gaps
- Build a remediation plan
- Produce an audit-ready conclusion

## 5. Frameworks Used
- NIST SP 800-53
- ISO 27001
- CIS Controls
- SOC 2
- UK GDPR
- Cyber Essentials

## 6. Key Risks Assessed
- Leaver access
- Privileged access
- Supplier access
- Excessive permissions
- Poor access review evidence
- Unapproved role changes
- Weak log monitoring

## 7. Deliverables
- Excel control mapping workbook
- Risk-control matrix
- Evidence log
- Gap assessment
- Remediation tracker
- Short assurance report
- Optional process diagram

## 8. Key Findings
Summarise the top access governance gaps identified during the review.

## 9. Remediation Roadmap
Summarise priority remediation actions and control owners.
